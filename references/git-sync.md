# Git 自动同步详细配置

本文档包含 sg-task Git 自动同步功能的详细配置说明。

## 配置文件

**配置文件位置：** `~/.claude/sg-task/config.yaml`

### 配置选项

```yaml
# 是否自动提交（默认: true）
auto_commit: true

# 是否自动推送（默认: true）
auto_push: true
```

## 提交信息格式

使用 emoji 前缀的提交信息格式：

| 场景 | 提交信息格式 |
|------|-------------|
| 修改文档 | `📝 auto: 更新 [文件名] - [简短描述]` |
| 创建文档 | `✨ auto: 新建 [文件名]` |
| 完成任务 | `🎉 auto: 完成任务 [任务名称]` |
| 添加仓库 | `➕ auto: 添加仓库 [仓库名]` |

## 错误处理

| 错误场景 | 处理方式 |
|---------|---------|
| 不是 Git 仓库 | 静默跳过，不影响正常功能 |
| 未配置远程仓库 | 仅提交本地，不推送 |
| 推送失败 | 提示 `⚠️ Git 推送失败，变更已提交到本地`，但不中断操作 |
| 网络不可达 | 提示 `⚠️ 网络不可达，变更已提交到本地`，继续执行 |

## 手动控制命令

### 立即同步

```bash
/sg-task sync
```

立即同步任务文档到 Git 仓库。

**输出示例：**
```bash
🚀 正在立即同步到 Git...
✅ 已提交 3 个文件更改
✅ 已推送到远程仓库
```

### 查看待提交变更

```bash
/sg-task status
```

查看当前未提交的文档变更。

**输出示例：**
```bash
📋 当前状态：
📝 待提交的变更：
- M 2026-02-10_对接支付宝转账产品/development.md
- M 2026-02-10_对接支付宝转账产品/product.md
```

## 自动提交触发场景

以下操作会自动触发 Git 提交（当 `auto_commit: true` 时）：

1. 使用 Edit 工具修改任务文档
2. 使用 Write 工具创建或更新任务文档
3. 执行 `/sg-task doc` 创建文档
4. 执行 `/sg-task progress` 更新进度
5. 执行 `/sg-task complete` 完成任务
6. 执行 `/sg-task add-repo` 添加仓库
7. 执行 `/sg-task remove-repo` 删除仓库

## 确认消息格式

Git 提交成功后，使用简洁的确认消息：

```
✅ 已自动提交到 Git
```

## 完全手动模式

如果只想手动控制提交，设置：

```yaml
auto_commit: false
auto_push: false
```

然后使用 `/sg-task sync` 命令手动同步。
